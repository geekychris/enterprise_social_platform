🔑 GlobalId System


64-bit ID layout: 
[8-bit type][16-bit tenant][40-bit sequence]
|  entity  |   tenant   |     unique ID      |

  • Type extraction — bits 56-63 encode entity type (USER, POST, MESSAGE, ...)
  • Tenant isolation — bits 40-55 encode tenant; queries physically cannot cross tenants
  • Sequence — 40 bits = ~1 trillion IDs per type per tenant
  • Single column — no composite keys; efficient JOINs & indexes

public static long create(EntityType type, int tenantId, long seq) {
    return ((long) type.code() << 56)
         | ((long) tenantId << 40)
         | seq;
}
public static int extractTenant(long globalId) {
    return (int) ((globalId >> 40) & 0xFFFF);
}

Querying with Trino


-- Daily active users by tenant
SELECT tenant_id,
       event_date,
       COUNT(DISTINCT user_id) AS dau
FROM warehouse.user_activity_hourly
WHERE event_date >= CURRENT_DATE - INTERVAL '30' DAY
GROUP BY tenant_id, event_date
ORDER BY event_date DESC;

-- Top posts by engagement
SELECT p.id, p.content_preview,
       COUNT(DISTINCT r.user_id) AS reactions,
       COUNT(DISTINCT c.id) AS comments
FROM warehouse.posts_all p
LEFT JOIN warehouse.reactions_all r ON r.post_id = p.id
LEFT JOIN warehouse.comments_all c ON c.post_id = p.id
WHERE p.tenant_id = 1
  AND p.created_date = CURRENT_DATE
GROUP BY 1, 2
ORDER BY reactions + comments DESC
LIMIT 20;

📨 Event Delivery


graph LR EV["Event Emitted"] --> Q["Per-App Queue"] Q --> D["Dispatcher"] D -->|POST| WH["App Webhook URL"] WH -->|200 OK| DONE["Delivered ✓"] WH -->|5xx| RT["Retry Queue"] RT -->|exp backoff| D RT -->|max retries| DL["Dead Letter Queue"] style EV fill:#4f46e5,stroke:#4f46e5,color:white style DONE fill:#10B981,stroke:#10B981,color:white style DL fill:#dc2626,stroke:#dc2626,color:white

  • Per-app queues — one slow app doesn't block others

  • Dead letter queue — inspect and replay failed events
